What are the basic git commands? Talk about how git is primarily a *command line* thing, although clients exist.
git config --global user.name / user.email Tells git who you are, shows up in logs
git init Starts a repository. From empty, or existing.
git clone Creates a new repository from someone else's repository.
git add adds a file to a git repository. Ask what situations might you *not* want to add a file to a repo? (passwords, etc)
git commit "Saves" your added files, makes log entry
git remote Defines a remote repository. Can have multiple, local, network, with different names.
git push push changes to a remote repo
git status
git checkout Allows you to go BACK in TIME, or to different branches, or individual files, etc. Incredibly powerful and very confusing at times
git branch This is what makes git, git. Try to describe cheap branching. If anyone has used svn or other SCMs, how was their experience with it, etc.
git pull Incorporates changes from remote repo into your own -- for collaboration, etc.
git diff Shows differences -- useful for determining what has changed across commits.
git log What happened when? Note how each commit has its own hash -- security and integrity!
git merge Make changes on an experimental branch, want to merge them into a current?
